# 협업 툴 인식 강화 — 제품·기술 방향 (실행 문서)

CSTalk/ECH를 **메신저가 아니라 협업 허브**로 인식하게 만드는 기능 방향을, 개발·운영이 동일한 기준을 쓰도록 정리한 문서입니다.  
(첨부 플랜의 할 일 항목을 이 저장소에서 **문서·로드맵 항목**으로 이행했습니다. 플랜 원본 파일은 수정하지 않습니다.)

---

## 1) 우선 사용자 여정: 멘션 → 업무 생성 → 마감 → 칸반 이동

### 여정 정의

1. 사용자 A가 채널/DM에서 B를 **@멘션**한다.  
2. B가 맥락을 보고 **메시지에서 업무 항목**을 만든다.  
3. B가 업무에 **마감·우선순위**를 두고 추적한다.  
4. B(또는 팀)가 동일 맥락에서 **칸반 카드**로 옮겨 상태를 보드에서 관리한다.

### 현재 구현(끊김 없는 구간)

| 단계 | 상태 | 근거 |
| :--- | :--- | :--- |
| 멘션 | **있음** | `Phase 2-6`, 멘션 알림·토스트·채널 이동 |
| 메시지 → 업무 생성 | **있음** | `POST /api/messages/{id}/work-items`, 메시지–업무 링크 |
| 칸반(보드·카드·담당·이동) | **있음** | 채널 연동 칸반 API·워크 허브 UI |

### 기능 갭(끊기는 지점) — 우선순위 순

1. **마감·우선순위 부재**  
   - `work_items`에 **마감일(`due_at`)·우선순위** 필드가 없고, API·워크 허브에도 일관된 입력·표시가 없다.  
   - **영향:** “언제까지”가 대화와 분리되지 않아 협업 툴 인식이 약해진다.

2. **채널 무관 “내 할 일” 뷰 부족**  
   - 사이드바 **내 업무**는 칸반 **담당 카드** 기준으로 유용하나, **멘션된 메시지에서 만든 업무**·**마감 임박**을 한 화면에서 묶는 **오늘/지연/멘션 연계** 뷰는 없다.

3. **마감·지연 알림이 업무 단위로 부재**  
   - OS/브라우저 알림은 메시지·멘션 중심이다. **업무 마감 D-1 / 지연**을 업무 단위로 밀어주는 정책·API·UI가 없다.

4. **멘션 → 업무 → 칸반 원흐름 UI**  
   - 허브 내에서 생성·저장은 가능하나, **멘션 토스트에서 곧바로 업무 생성**, **업무 행에서 칸반 컬럼으로 배치** 같은 **단일 여정 최소 클릭**은 아직 약하다.

5. **상태 단일 진실(메시지 vs 보드)**  
   - 업무 `status`와 칸반 카드 위치·상태가 **자동 동기화**되지 않는다(의도적 분리일 수 있으나, 사용자 기대와 어긋나면 “끊김”으로 느껴짐). 정책 문서화·선택적 동기화가 후속 과제.

## 2) 문서·지식 축 — 제품 방향 확정 (1안)

후보: **채널 자료실** · **위키/페이지** · **노트**

### 확정 (1차)

- **1차 축: 채널 고정 자료실**  
  - 이미 **파일 업로드·파일 허브·첨부 묶음**이 있으므로, 동일 도메인을 **채널 단위 “거주지”**로 확장하는 것이 비용 대비 인식 전환 효과가 크다.  
  - 구체 목표(로드맵 `Phase 2-7-2`로 분해): **폴더/핀**, **설명·태그**, **“이 메시지의 첨부 모음” 링크**, (선택) **간단 버전/교체 이력 메타**.

### 2차 (별도 Phase로 분리)

- **위키/페이지형 지식**: 협업 스위트 인식에 강력하나 스키마·에디터·권한 모델이 크다. **자료실 1차 안정화 후** `ROADMAP`에 별도 Phase로 추가한다.  
- **개인 노트 전용 제품**: 본 제품 축과 겹치면 범위 폭발 → **우선 제외**. 필요 시 “나에게 쓰기 DM + 업무”로 최소 충족.

## 3) 캘린더 범위 (MVP + AI 연동 전제) — 로드맵 정합

| 플랜 항목 | 저장소 상태 |
| :--- | :--- |
| 1인 1달력·직접 일정 | **구현됨** — `Phase 6-1`, `calendar_events` |
| 채널·DM 수락형 공유·복사본 | **구현됨** — `Phase 6-2`, `calendar_share_requests` |
| 출처·감사 | **구현됨** — `Phase 6-3`, `origin_channel_id`, `AuditEventType.CALENDAR_*` |
| `CalendarSuggestion`·확정 API 통일·`originMessageIds`·바쁨/충돌 API | **미구현** — `Phase 6-4` |
| iCal 가져오기/보내기 | **미구현** — `Phase 6-5` |
| AI가 타인 일정 쓰기 | **원칙 확정** — 반드시 **공유 → 수락** 경로만 (코드·프롬프트 설계 시 동일) |

개발 원칙(플랜과 동일): 제안(초안)과 확정은 **분리**하고, 확정은 **사용자 직접 생성과 동일 API**를 타게 설계한다.

## 4) AI 비서 1차 범위 + 캘린더 도구 (설계 기준)

### 대화·검색 축

- 채널/스레드 **요약**(권한 내 컨텍스트만).  
- **답장 초안**(톤·길이 선택, 전송 전 편집 기본).  
- 통합 검색 결과를 근거로 한 **Q&A**(인용 필수).

### 실행 축 (기존 API 결합)

- 자연어 → **업무 초안** (`메시지→업무` API와 결합).  
- 자연어 → **칸반 카드 초안**(자동 이동·담당 변경은 **제안만**, 확정은 사람).

### 캘린더 도구(백엔드 API를 먼저 고정)

- 권한 범위 내 **내 일정 조회**(제목 마스킹 옵션 포함 가능).  
- **충돌 검사** / 바쁨 구간 조회.  
- **제안 생성·취소**(`CalendarSuggestion` 등) — 달력 반영은 **확정 API 한 경로**.

### UX 공통

- 캘린더·업무 관련 AI 출력은 **인라인 스팸 대신 제안함(Inbox)** 큐에 적재하는 패턴을 기본으로 한다.

## 5) AI 거버넌스 (데이터 주권·경계)

### 기본 정책

- 채널·DM·첨부 **원문**은 **공용 인터넷 LLM**으로 **기본 전송하지 않는다**.  
- 게이트웨이에서 **기본 차단**; 예외는 **온프레/전용 VPC/계약상 미학습·보존 배제**가 명시된 경로만 **법무·보안 합의** 후 허용.

### 구현 시 체크리스트

- 요청/응답 **메타 감사**(채널 ID·액터·모델 식별자·토큰 대략치 등; 본문은 정책에 따라 제외 또는 해시).  
- **PII 마스킹** 파이프라인(사번·계좌 등 금지 패턴).  
- **RAG/임베딩**: 권한 필터 이후만 인덱싱하거나, 외부 임베딩 시 **최소 청크·비식별** 별도 DPIA.  
- **환각 완화**: UI 기본값 = **근거 `message_id`(들)·편집 후 전송**.

## 6) 프로액티브 비서 정책

| 항목 | 정책 |
| :--- | :--- |
| 관찰 범위 | **채널 단위 옵트인** 권장; 워크스페이스 전역 기본 “항상 관찰”은 지양 |
| DM | **프로액티브 자동 관찰 제외** — 사용자 명시 요청(구간 선택·명령)만 허용 |
| 방해 최소화 | **제안함** 배지, **신뢰도 임계값** 미만은 적재하지 않음 |
| 상한 | 채널·스레드당 **시간당 제안 상한**, **거절 후 쿨다운** |
| 다이제스트 | 실시간 대신 **일/주 단위** 묶음 옵션 |
| 변이(mutating) 작업 | **사용자 확인 후**만 실행(업무 생성·일정 확정 등) |
| 트리거 | **규칙·소형 모델** 1차 필터 → 필요 시만 **짧은 컨텍스트 LLM** |

## 7) LLM 비용·쿼터·관측

### 통제

- **컨텍스트 캡**: 메시지 개수·문자 상한; 동일 구간 **윈도우 해시 캐시**로 중복 호출 방지.  
- **배치·지연**: N초 침묵·스레드 단위 묶음 호출.  
- **워크스페이스/일 단위 쿼터**: 초과 시 **규칙 전용 모드** 또는 제안 중단 + 관리자 알림.

### 관측 대시보드(권장 지표)

- 호출당·채널당 **토큰 in/out**, 모델명, **거절률**, 옵트인 채널 수, 큐 적체.  
- 임베딩은 **증분만** 처리해 비용 분리 기록.
